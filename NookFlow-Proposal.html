01 / 创意简介

把刘海屏变成任务的入口，而不是通知的终点

MacBook 的刘海是一块永远在最前方的视觉区域，今天它只用来显示时间。NookFlow 让这块区域与用户当前的会议或学习任务同步——开始任务时它展开为工作卡片，结束时它把过程沉淀为可回看的记录。

基础项目

X Nook 已实现 macOS 刘海工具中心：日历、便签、文件托盘、媒体控制、双指手势切换、主题与设置。但这些能力目前彼此独立，缺少围绕"一次任务"的串联。

赛期内迭代：NookFlow

NookFlow 不是原样提交旧项目，而是在 X Nook 之上新增"任务上下文工作流"内核。原来的日历、笔记、文件被整合进一个以任务为单位的生命周期：选择日程 → 开始任务 → 记录与暂存 → 结束沉淀 → 回看。

02 / 目标用户与痛点

反复切换应用，是注意力的隐形杀手

核心用户为使用 MacBook 的学生与办公人群。他们在上课、开会或深度学习时，需要频繁在日历、笔记、Finder 与多个应用之间切换，临时资料难以与具体任务关联，结束后难以回溯。

切换成本高

日历看时间、笔记记内容、Finder 翻资料、聊天工具对日程——一次任务要打开 4-5 个应用，注意力被反复打断。

资料与任务脱节

会议中临时下载的文件、随手记的要点，散落在桌面和多个笔记里。一周后想找"上次那个会议的设计稿"，往往要翻半小时。

缺少任务边界感

没有明确的"开始 / 结束"仪式，任务之间的上下文互相串味。结束会议后没有沉淀环节，待办和关键决策容易被遗忘。

04 / 核心功能

NookFlow 在赛期内要新增与整合的能力

以下为参赛版本规划的核心功能。其中"任务上下文工作流"是 NookFlow 相对 X Nook 的主要新增内核；日历、笔记、文件托盘为已有能力的整合升级。

任务上下文卡片

从日程进入任务后，刘海展开为上下文卡片：日程标题、倒计时、关联笔记入口、文件暂存区、待办勾选。所有动作集中在一块区域。

日程驱动入口

日历 Widget 不再只是查看，而是任务入口。选中即将开始的日程即可一键进入任务上下文，日程元数据（时间、地点、参与人）自动带入卡片。

关联笔记

每个任务自带一份笔记，记录的内容自动绑定到该任务。无需新建文档、命名、归类——任务结束，笔记就位。

文件暂存托盘

会议中拖入的文件自动绑定到当前任务，结束时一并沉淀。一周后回看任务，原始文件、笔记、待办一起出现。

任务待办清单

任务卡片内置轻量待办，记录当场确认的事项与负责人。结束任务时未完成项可标记为延续到下次。

任务回看

结束的任务沉淀为历史记录，可按时间或日程查看。每条记录包含笔记、文件、待办完成情况，形成可追溯的工作日志。

05 / 价值与意义

让任务有边界，让上下文可回看

NookFlow 的价值不在于"又一个笔记应用"，而在于把分散的工具按任务串联起来，给学生和办公人群一个低成本的上下文沉淀方式。

−4

减少应用切换

从日历、笔记、Finder、聊天工具的反复跳转，收敛到刘海一块区域。注意力被打断的次数显著下降。

1:1

资料与任务绑定

文件、笔记、待办不再是孤儿，而是天然属于某次会议或某节课。回溯时无需在文件夹里大海捞针。

∞

可回看的工作日志

每一次任务都沉淀为一条记录。学生可以复习"那节课记了什么"，办公用户可以追溯"那次会决定了什么"。
