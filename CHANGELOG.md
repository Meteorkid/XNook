# Changelog

## v1.3.0 (2026-07-05)

本版为 **跨岛联动与设置收口** 版本。相对 v1.2.1，X Nook 不再只是一个单体灵动岛，而是补齐了与 X Island 的切换协议、启动显示仲裁和更多可见可调的日常设置，同时把媒体、文件托盘、图标与核心交互再打磨了一轮。

### 相对 v1.2.1 的重点变化

| 维度 | v1.2.1 | v1.3.0 |
|------|--------|--------|
| **灵动岛联动** | 只能各自显示，缺少稳定切换 | 新增双指横滑切换、URL Scheme 联动、收起态窗口归属判断、启动显示仲裁 |
| **个性化显示** | 主题与默认值分散，运行态和设置页不一致 | 新增系统 / 浅色 / 深色主题，统一默认值注册，设置页与实际行为保持一致 |
| **提醒与权限** | 日历只负责显示 | 新增日历提醒音、提前时间、静音时段、权限快捷入口 |
| **媒体与内容** | GIF、歌词、文件托盘能力较基础 | 新增多 GIF 管理与持久化、歌词兜底优化、托盘拖入拖出与高亮反馈 |
| **更新与诊断** | 版本信息可见性较弱 | 新增更新检查状态、最新版本信息、打开发布页入口 |

### 联动与切换

- 新增 **双指左右滑动切换 X Nook / X Island**，仅在收起状态生效，避免和展开面板中的滚动手势打架。
- `AppSwitcher` 改为基于 URL Scheme 的跨进程唤醒链路，减少单纯激活应用后两个岛同时悬浮的错位感。
- 新增 **收起态窗口归属判断**，只让当前位于最上层、且处在手势命中区域的岛响应横滑。
- 启动阶段加入 **显示仲裁**，支持“上次使用 / 指定默认岛”这类偏好，不再默认两个应用一起抢占灵动岛区域。
- 设置页新增联动面板，可直接查看对方应用是否已安装、是否运行、协议是否就绪，并提供“立即切换测试”按钮。

### 设置与外观

- 新增 **系统 / 浅色 / 深色** 主题切换，X Nook 的主窗口和设置窗口会跟随同一套主题状态。
- 引入 `SettingsDefaults` 作为统一默认值来源，修复“启动默认值”和“设置页文案默认值”不一致的问题。
- 对交互、显示、提醒、联动等分组重新收口，设置页不再把不同层级的偏好混在一起。
- 保留宽而矮的 X Nook 面板尺寸取向，同时让默认宽高、Ticker、歌词开关与运行时实际行为一致。

### 媒体、托盘与提醒

- 新增 **多 GIF 管理**：支持添加、切换、删除多个 GIF，并将文件复制到 Application Support 持久化保存。
- 文件托盘补齐 **拖入自动展开、拖出到 Finder、滚动列表、高亮反馈、右键菜单和删除入口**。
- 媒体侧继续优化歌词与脚本桥接稳定性，减轻曲目切换时的状态丢失和显示延迟。
- 新增 **日历提醒音**、提醒提前量、静音时段和试听按钮；同时提供跳转日历权限、自动化权限和登录项设置的系统入口。

### 更新、图标与质量

- 新增 **检查更新** 状态流，包括最新版本号、上次检查时间和打开 GitHub Release 页入口。
- 应用图标更新为新的品牌版本，配套修正应用包内图标资源。
- 为灵动岛联动、媒体、日历、脚本桥接等关键路径补充测试，降低后续继续打磨时的回归风险。

---

发布建议：如果你已经在同时使用 X Nook 与 X Island，升级到 v1.3.0 后先到“联动”面板确认默认显示岛与手势灵敏度，再按自己的使用习惯调整主题、提醒音和默认展开行为。
