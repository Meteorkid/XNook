# NookFlow 最近任务显示条数与自适应高度设计

## 目标

让用户在 X Nook 设置中选择最近任务的显示条数（1–5 条，默认 3 条），并使展开面板的高度随实际显示的任务行数变化。

## 范围

- 在设置窗口的“显示”页新增“最近任务显示条数”选择器。
- 将选择持久化到 `UserDefaults`，未设置时使用 3 条。
- 最近任务区域按所选条数显示历史记录；历史记录本身不裁剪、不删除。
- NookFlow 内容高度使用与 SwiftUI 列表一致的确定性布局常量：每增加一条显示记录，面板增加一条记录行的高度及既有间距。
- 展开面板按完整内容高度展开；窗口边界由 `NotchWindow` 在屏幕范围内保护，不压缩 Widget。

## 不在范围内

- 不增加历史记录详情页、滚动列表或删除历史记录功能。
- 不改变活跃任务卡片、日历、笔记或文件架逻辑。

## 实现边界

1. 使用单一设置键 `nookFlowHistoryDisplayLimit`，合法值为 1–5。
2. `FocusSessionView` 接收当前显示上限，并使用它裁剪视图列表。
3. `NotchContentView` 监听该设置和 NookFlow 高度变化，重新请求窗口尺寸；任务行高度仅由 `FocusSessionView` 提供，避免重复硬编码。
4. 移除 `IslandSizeCalculator` 的固定高度裁剪；由 `NotchWindow` 的屏幕边界保护窗口尺寸。

## 验收与测试

- 默认显示 3 条；设置 1、2、3、4、5 条均只显示对应数量的最新记录。
- 修改条数后，展开面板无需重启应用即可变高或变矮。
- 一条记录对应一条可见任务行的高度增量，且头部、上下内边距不重复累加。
- 条数增加时，任务区域与 Widget 都保持完整可见；窗口不会超过屏幕边界。
- 现有 NookFlow 会话持久化与测试不得回归。
